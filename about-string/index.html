rust の String をよく知らなくてコンテストで爆死したお話

2020-05-18

事の発端

それは abc168-A で起きた出来事でした。
一の位で場合分けをすれば良いとのことで、私は普通に次のコードを書きました。

let s: String = read_string();
match s[s.len()-1] {
    ...
}

C++ をやっていた人間であれば、こんな感じのコードを書くはずです。

私はなんの疑いもなくウキウキでcargo runをしましたが・・・通りません

は？？？？？？？？？？？？？？？？？？？cargo バグったか？？？？？？？？？？？
と思いながらターミナル再起動したりcargo run連打したりしましたが、通りません

思わず台パンをしてしまいましたが、落ち着いてからエラーメッセージを読むとこんなエラーメッセージが出ていました。

match s[s.len()-1 ]{
    　^^^^^^^^^^^^^ `std::string::String` cannot be indexed by `usize`

ふむふむ、どうやら String の添え字アクセスはできないようですね。 こうゆうときに限って解決法書いてくれない rustc 本当に嫌いです。どうすればええんや・・・

結論

結論から言うと、Vec に変換してあげれば ok です。

let s: String = read_string();
let s: Vec<char> = s.chars().collect();
match s[s.len()-1] {
    ...
}
