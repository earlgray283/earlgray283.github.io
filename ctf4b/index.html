Beginner's CTF 2020 Writeup

2020-05-24

Beginner's CTF 2020 に参加してきました。

　Beginner's CTF に rm -rf /kadai として参加しました。メンバー全員が CTF 初参戦でしたが、217点を獲得し361位となることができたのでまあよかったんじゃないかなと思います。個人的には pwn が解けなかったのが一番悔しいです。Conglaturation! じゃあないんだよな。
　私は Reversing の mask, Misc の emoemoencode, Pwn の Beginner's Stack に挑戦したのでそれらについて書いていこうかなと思います。


mask(Reversing, 62pt)

　一番最初に解いた問題です。
まずは mask のバイナリファイルを落としてファイルの確認をすると ELF ファイルだったのでまあ実行するんだろうなと思って実行したらなにか出てきました(ここまで早口)

$ ./mask
Usage: ./mask [FLAG]

　ということで適当なフラグを入れて見たけど、わからん！ｗ

$ ./mask ctf4b{329}
Putting on masks...
atd4`q101u
c`b bk#")i
Wrong FLAG. Try again.

　バイナリファイルが与えられたら strings で中身見ろっておじいちゃんが言っていたのを思い出し、確認をしてみました。すると、Putting on masks...の付近になにやら見慣れた文字列が・・・

$ strings mask
.
.
Putting on masks...
atd4`qdedtUpetepqeUdaaeUeaqau
c`b bk`kj`KbababcaKbacaKiacki
Correct! Submit your FLAG.
.
.

　ふむふむ、つまり変換後の文字列がatd4`qdedtUpetepqeUdaaeUeaqau->c`b bk`kj`KbababcaKbacaKiackiというように変換されるように入力を与えてあげればいいということですね。
　ということで、a-z A-Z それぞれがどのように変換されるかを調べ、条件を満たすように全パターン列挙するコードを書きました！競プロで培ってきた†全探索†です。(変数名クッソ雑ですみません・・・)

#include<stdio.h>
#include<string.h>
int main() {
    char *alphabet = "abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ";
    char *list1 = "a`adede`a`adedepqpqtutupqpA@ADEDE@A@ADEDEPQPQTUTUPQP";
    char *list2 = "abc`abchijkhijk`abc`abchijABC@ABCHIJKHIJK@ABC@ABCHIJ";
    char *ans1 = "atd4`qdedtUpetepqeUdaaeUeaqau";
    char *ans2 = "c`b bk`kj`KbababcaKbacaKiacki";
    char ans[1000];
    for(int i=0;i<strlen(ans1);i++)ans[i]='.';
    ans[strlen(ans1)]='\0';
    for(int i=0;i<strlen(ans1);i++){
        for(int j=0;j<strlen(list1);j++) {
            if(ans1[i]==list1[j]&&ans2[i]==list2[j]){
                ans[i]=alphabet[j];
                printf("> %s\n", ans);
                break;
            }
        }
    }
}

これの出力結果を mask の引数に渡すと合ってるよ！って言ってくれました。


emoemoencode(Misc, 53pt)

　google 先生に聞いても知らないの一点張りだったので自分でなんとかしました。
文字は数字！と毎日唱えていればそこまで難しくはありません。最終的にどうせctf4b{...}って形になるんだろうな～って思いながら🍣を 'c' と考え、あとは文字の足し算引き算を頑張って、すべての絵文字を半角英字に置き換えました。そしたらフラグが出てきたのでニッコリしました。
